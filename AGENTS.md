# Project Instructions

This is a mobile-first Life RPG app.

Important rules:
- Do not rewrite the architecture unless absolutely necessary.
- Audit the existing code before making big changes.
- Protect the core loop:
  Mission -> Focus -> Proof -> Judge -> Reward -> Visible Upgrade
- Treat auth, rewards, purchases, admin actions, and progression as high-risk.
- Always run the smallest relevant checks before finishing work.
- Do not mark work complete without evidence.
- Prefer small, reviewable changes.
