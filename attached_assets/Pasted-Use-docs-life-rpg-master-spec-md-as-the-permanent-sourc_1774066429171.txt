A. CONTROLLED API SURFACE LITE
==================================================

Add a small, safe API layer for key product capabilities.

Purpose:
Allow safe programmatic access to the most useful parts of the system without opening everything.

Possible API surface areas:
- read current profile summary
- read current skills / arc / title summary
- read active missions / AI missions
- create mission
- update mission state in controlled ways
- read reward/wallet summary
- read selected share/progress summaries
- trigger selected internal flows if safe

Requirements:
- keep APIs controlled and minimal
- use authentication and ownership checks
- if API keys/tokens are added, keep them scoped and revocable
- preserve JSON-only contracts
- do not expose sensitive raw data by default
- do not expose unsafe mutation surfaces
- prefer read-heavy and carefully limited write surfaces

Rules:
- this is not a public free-for-all API
- keep schema and contract clean and stable
- only expose what is already mature and safe
