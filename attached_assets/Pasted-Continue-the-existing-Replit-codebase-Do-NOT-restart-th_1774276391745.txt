PART 3 — INTEGRATION
==================================================

3A. Rewards Screen Game Mode Buttons

Wardrobe → navigates to Wardrobe screen
Garage → navigates to Garage screen
Both must work without crashes

3B. Character Integration

Equipped wardrobe items must affect character:
  Watch → wrist layer in CharacterRenderer
  Top → outfit layer in CharacterRenderer
  
When item equipped:
  CharacterRenderer receives new props
  Visual updates smoothly

3C. Coin Gate Enforcement

ALL purchase logic server-side:
  POST /api/marketplace/buy
  Checks: owns item? level met? coins sufficient?
  Deducts coins atomically
  Adds to inventory
  Returns updated state

Never trust client-side purchase signals.

==================================================
PART 4 — ITEM STATES CONSISTENCY
==================================================

Across ALL screens (Wardrobe, Garage, Shop):

Use same language always:
  LOCKED = level requirement not met
  AVAILABLE = level met, can afford
  AFFORDABLE = level met, cannot afford yet
  OWNED = purchased/auto-owned
  EQUIPPED = currently active

Use same visual chips:
  LOCKED: 🔒 + "Lvl X" — amber chip
  AVAILABLE: price — accent chip
  AFFORDABLE: price in red — muted chip
  OWNED: "Owned" — green chip
  EQUIPPED: "Equipped ✓" — accent border
