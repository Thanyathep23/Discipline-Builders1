==================================================
A. PRODUCT CATALOG DATA MODEL
==================================================

Build a proper catalog-driven item/product system.

Purpose:
Allow the product/store to scale without hardcoding individual items or item cards.

Each product/item should support at minimum:
- id
- name
- slug
- short description
- full description if useful
- product type
- category
- subcategory
- tags
- rarity
- status / availability state
- price(s) or acquisition mode
- currency type if relevant
- premium flag if relevant
- limited/exclusive flags if relevant
- display/equip eligibility
- world/status/profile relevance flags if useful
- preview/media references
- visibility order / merchandising priority
- created/updated timestamps

Suggested product types:
- title
- badge/trophy
- cosmetic
- room/world item
- accessory
- watch
- jewelry
- vehicle
- command-center upgrade
- lifestyle item
- prestige marker
- premium content unlock
- event item
- utility-style identity item

Rules:
- keep schema normalized but practical
- do not overbuild polymorphism
- make the system easy to extend

==================================================
B. CATEGORY / SUBCATEGORY / TAG SYSTEM
==================================================

Add a clean merchandising taxonomy.

Minimum requirements:
- top-level categories
- optional subcategories
- tags for flexible grouping and merchandising

Recommended top-level categories:
- Watches
- Cars
- Jewelry
- Fashion / Style
- Room / Command Center
- Prestige / Identity
- Lifestyle
- Trophies / Badges
- Limited / Event
- Premium / Exclusive

Requirements:
- categories should work both in admin and storefront
- support filtering and sorting
- tags should allow lightweight campaign/season/arc/category cross-grouping
- avoid taxonomy bloat

==================================================
C. ADMIN MERCHANDISING WORKFLOW
==================================================

Build the minimum admin flow to add and manage products properly.

Admin must be able to:
- create product
- edit product
- archive / deactivate product
- set category / subcategory / tags
- set price / acquisition mode
- upload or assign preview/media
- set rarity / exclusivity / limited flags
- mark whether the product is:
  - purchasable
  - premium-only
  - unlock-only
  - displayable in room/world
  - displayable in profile/status
  - equippable
- control visibility order / featured state
- optionally connect product to:
  - event
  - content pack
  - season/cycle
  - prestige/mastery gate
  - arc relevance
  - admin-only test state if needed

Requirements:
- admin-only
- validation must be server-side
- all admin changes should be audit-logged where appropriate
- keep UI tactical and efficient
- do not build a bloated merchandising suite

  - owned/equipped state if relevant

Rules:

==================================================

- expired
- archived

- premium-only
- limited

==================================================

  - premium entitlement checks

Do not over-engineer, but do make the system trustworthy.
