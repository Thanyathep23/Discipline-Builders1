==================================================
A. AI LIFE PROFILE
==================================================

Add a new layered onboarding/profile system that maps real life into the game.

The onboarding must be split into 3 layers:

1. Quick Start
Used to start the game fast.
Collect only:
- current main goal
- main life problem right now
- work/study status
- available time per day
- areas the user wants to improve
- strictness preference

2. Standard Profile
Can be completed after starting.
Collect:
- daily routine
- weak points
- distraction triggers
- current habits
- sleep pattern (basic)
- health/fitness status (basic)
- money/finance situation as RANGE only, not exact balance
- confidence in different life areas

3. Deep Profile
Optional and resumable later.
Collect:
- longer-term goals
- constraints
- environment/support system
- self-described strengths/weaknesses
- life context useful for personalized mission generation

Requirements:
- User can pause and continue later
- Each question should explain why it is being asked
- User can skip optional questions
- User can edit profile answers later
- Do NOT require exact bank balance, exact address, or invasive relationship history by default
- Keep privacy-safe defaults

Suggested new tables / schema additions if missing:
- life_profiles
- life_profile_sections
- life_profile_answers
- onboarding_progress

Need backend ownership checks for all profile data.
