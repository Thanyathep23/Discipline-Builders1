H. BACKEND / SECURITY RULES
==================================================

- All skill updates must happen server-side
- All AI mission generation persistence must happen server-side
- All proof requirement persistence must happen server-side
- Users can only access their own profile/skills/missions/inventory unless admin
- Do not trust client-provided skill XP or rewards
- Keep reward calculation server-side only
- Keep proof verdict finalization server-side only
- Add ownership checks for all new routes
- Return JSON from all API routes
- Add minimal validation for all new inputs
- Avoid introducing insecure sensitive data handling

==================================================
I. FALLBACK STRATEGY
==================================================

If OpenAI is unavailable:
- profile interpretation should still work with rule-based defaults
- AI mission generation should still work with template + rules
- proof requirement generation should still work from mission type + difficulty
- do not block the product

The product must remain usable without an API key.
