Goal:
Implement Phase 27 — Game Mode Foundation / Character Status Hub.

Current context:
- The existing app already has the core Life RPG systems: missions, proof, rewards, skills, identity, marketplace, room/world, premium, community, and guidance systems.
- The next expansion is a dedicated mobile-first “game mode” layer.
- This game mode should make the user feel like they are looking at a real player character whose life is evolving visually over time.
- The immediate goal is NOT to build a full 3D game.
- The correct goal is to build a premium pseudo-3D / rendered-character status hub that is realistic to implement in Replit and future-ready for App Store deployment.

Your task:
Build only the smallest high-impact Game Mode Foundation / Character Status Hub layer.

==================================================
PHASE 27 SCOPE ONLY
==================================================

Build only:

1. Game Mode entry point and navigation
2. Character Status Hub screen
3. Starter character presentation
4. Mobile-first status layout
5. Core stat visualization for the 4 key dimensions
6. Room preview starter state
7. Basic progression summary / next unlock hint
8. Minimal connections to existing systems
9. Minimal internal/admin controls only if strictly needed

Do NOT build:
- full 3D interactive character
- full 3D world
- complex animation system
- freeform avatar editor
- clothing equip system in this phase
- car collection UI in this phase
- full room decorator in this phase
- player discovery / matchmaking
- social comparison systems
- redesign of the whole app

STEP 1 — AUDIT FIRST
==================================================

Before changing anything:
1. inspect the current mobile app structure, profile/status screens, home/command center, world/room layer, marketplace, character/status visualization work, and existing navigation

4. implement only the minimum systems needed
5. avoid broad refactors

==================================================
A. CORE PRODUCT FANTASY FOR THIS PHASE
==================================================

This phase must create the first true “game mode” experience.

The user should feel:
- this is my character
- this character reflects my current life progress
- I started from a humble/basic state
- my body, presence, status, and environment will improve as I improve
- my room, clothing, accessories, and future collection goals will matter
- my daily actions are shaping a visible life-avatar

Important:
Do NOT use insulting or shaming labels like “poor” directly in the UI.
The visual starting state can feel humble/basic/starter-level, but the language must stay aspirational and premium.

Preferred tone:
- serious
- premium
- stylish
- life-upgrade fantasy
- not childish
- not cartoonishly goofy

==================================================
B. GAME MODE ENTRY POINT
==================================================

Add a clear Game Mode entry point inside the existing mobile app.

Requirements:
- users should be able to open a dedicated Game Mode / Character Hub screen
- this should feel like entering the “player identity” layer of the product
- integrate cleanly into the current app navigation without rewriting the whole nav system
- use current mobile app patterns where possible

Possible naming directions:
- Game Mode
- Character
- My Avatar
- Status Hub
- Identity Hub

Choose the most premium and coherent naming based on the existing app tone.

==================================================
C. CHARACTER STATUS HUB SCREEN
==================================================

Build the main Character Status Hub screen.

This is the hero screen of this phase.

The screen must show at minimum:
1. full-body stylized player character
2. current status tier/class
3. 4 major progression dimensions:
   - Fitness
   - Discipline
   - Finance / Lifestyle
   - Prestige
4. short explanation of what those dimensions currently say about the player
5. room preview starter card/section
6. next unlock / next evolution hint
7. quick action buttons or links to:
   - improve character/status
   - open store
   - view room
   - view collection (placeholder/entry for future car phase if needed, but do not build the full feature now)

Requirements:
- mobile-first layout
- strong visual hierarchy
- character is the hero of the page
- stats support the fantasy, not overpower it
- page should feel screenshot-worthy
- page should be understandable quickly

==================================================
D. STARTER CHARACTER PRESENTATION
==================================================

Build the first starter version of the player character.

Mandatory starter look:
- white shirt
- black jeans
- clean simple shoes or neutral starter footwear
- humble/basic starter vibe
- not broken/dirty/exaggerated poverty
- more like “beginning stage” than “humiliation stage”

Character style:
- stylized
- pseudo-3D / rendered
- premium
- slightly aspirational
- not too childlike
- not uncanny realism
- full body visible

Implementation direction:
- do NOT build a full 3D model system in this phase
- use a pseudo-3D / rendered-character presentation that is realistic for Replit/mobile
- this can be based on layered illustration, composable assets, or a deterministic rendered status figure approach
- keep it lightweight and extensible for later phases

==================================================
E. CORE STATUS DIMENSIONS
==================================================

In this phase, visualize exactly these 4 dimensions:

1. Fitness
Represents:
- body condition
- posture
- health/energy vibe
- physical confidence

2. Discipline
Represents:
- self-control
- consistency
- composure
- neatness/refinement

3. Finance / Lifestyle
Represents:
- overall lifestyle quality
- taste/status refinement
- material progression signals
- future access to better presentation and purchases

4. Prestige
Represents:
- earned distinction
- elite markers
- long-term accomplishment
- visible advancement beyond basic level

Requirements:
- these should be shown clearly on the Character Status Hub
- each dimension should have:
  - current value/state
  - short plain-language explanation
  - directional sense of growth if already practical
- do not overload the page with too many secondary stats

==================================================

Examples:
