"Cannot find native module 'ExpoGL'"

CharacterViewer3D.tsx uses @react-three/fiber 
which requires expo-gl native module.
expo-gl is NOT available in Expo Go.
This approach will never work in Expo Go.

Fix:
Remove all @react-three/fiber and expo-gl 
dependencies from CharacterViewer3D.tsx entirely.

Replace with a high-quality SVG character system
that works in Expo Go without any native modules.

STEP 5 — 360° SWIPE ROTATION
==================================================

Implement rotation between 3 views using
PanGestureHandler (already installed):

ANGLE MAPPING:
0° → Front view (CharacterFrontSVG)
90° → Side view (CharacterSideSVG)
180° → Back view (CharacterBackSVG)
270° → Side view mirrored (CharacterSideSVG scaleX:-1)
360° → Front again

TRANSITION:
Between each 90° zone:
- Crossfade using React Native Animated
- Opacity: current view 1→0, next view 0→1
- Duration: 200ms
- Creates smooth rotation feel

GESTURE:
const panGesture = Gesture.Pan()
  .onUpdate((e) => {
    // Map x translation to 0-360 rotation
    const newAngle = (currentAngle + 
      (e.translationX / SCREEN_WIDTH) * 360) % 360
    setDisplayAngle(newAngle)
  })

SNAP BEHAVIOR:
On gesture end: snap to nearest 90° (0/90/180/270)
Use Animated.spring for snap animation
Duration: 300ms

VISUAL INDICATOR:
4 small dots at bottom of character area:
● ○ ○ ○ (front active)
○ ● ○ ○ (side active)

Each posture stage adjusts SVG transforms:

  shoulderWidth: scale(1.05) 
  chest: more forward
  stance: legs slightly wider

peak: commanding
  shoulderWidth: scale(1.10)
  head: lift (+2px)
  overall: more presence

These are SVG transform overrides
applied to the shoulder/torso group elements.
