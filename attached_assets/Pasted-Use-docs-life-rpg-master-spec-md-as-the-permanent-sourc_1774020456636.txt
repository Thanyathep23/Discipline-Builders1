==================================================
STEP 1 — AUDIT FIRST
==================================================

Before changing anything:
1. inspect current skills, titles, badges, arcs, chains, rewards, comeback systems, shareable surfaces, and admin/tuning tools

4. build only the minimum systems needed to create long-term retention and endgame meaning
5. avoid broad refactors

==================================================
A. MASTERY TIERS FOR CORE SKILLS
==================================================

Add a mastery layer above normal skill growth.

Current core skills:
- Focus
- Discipline
- Sleep
- Fitness
- Learning
- Trading

Build:
- mastery tiers or mastery milestones above the normal rank/level progression
- mastery should reflect sustained real behavior, not short-term spikes
- mastery should feel rare and meaningful

Examples:
- Focus Mastery I
- Discipline Mastery I
- Trading Mastery I

Requirements:
- mastery must be tied to real evidence over time
- mastery must not unlock from one lucky streak
- mastery should be visible in skill UI and character identity surfaces
- keep logic server-side
- keep implementation lightweight

==================================================
B. PRESTIGE / ASCENSION LITE
==================================================

Add a lightweight prestige-like system for advanced users.

Purpose:
Give users a reason to continue after reaching strong progress levels without wiping core identity in a frustrating way.

Design rules:
- do NOT hard-reset the user in a punishing way
- preserve earned identity/history
- prestige should feel like entering a higher league/chapter, not losing everything
- prestige should unlock new status, titles, cosmetics, or progression markers
- prestige should be optional

Possible examples:
- Ascension Tier I
- New prestige border/title
- Prestige-only badge/trophy
- Prestige mission category unlock
- Prestige profile marker

Requirements:
- keep prestige lightweight and meaningful
- do not create a giant rebirth system
- do not destroy current progress in a confusing way
- prestige should be based on real sustained progress thresholds

==================================================
C. LONG ARC PROGRESSION
==================================================

Extend arcs so they feel more like long-form personal story progression.

Current arcs already exist.
Now add lightweight arc progression states such as:
- beginning
- developing
- advanced
- completed
or equivalent stage model

Requirements:
- arc stages should reflect evidence over time
- arc should feel like a journey, not just a label
- arc progression should influence mission tone and identity surfaces where appropriate
- completed arcs can unlock new titles/badges/prestige markers
- keep arc progression stable and not noisy

==================================================
D. SEASONAL / CYCLICAL GOAL LAYER LITE
==================================================

Add a lightweight long-horizon goal layer.

This is not a full live events system.
Keep it simple.

Examples:
- 30-day Focus Season
- 21-day Recovery Sprint
- Trading Practice Cycle
- Learning Sprint

Requirements:
- a cycle should group multiple missions over time
- progress should feel larger than one mission or one chain
- cycles should support long-term retention
- users should see current cycle progress
- completion can unlock meaningful symbolic rewards

Rules:
- keep it lightweight
- do not build a giant scheduler
- do not overwhelm the user with too many parallel cycles

==================================================
E. LONG-TERM RETENTION SURFACES
==================================================

Add lightweight surfaces that show the user why they should keep going.

Examples:
- next mastery milestone
- current prestige readiness
- current arc stage
- current seasonal/cycle progress
- “next major unlock” card
- “you are close to...” card

Requirements:
- these surfaces should create anticipation
- should feel premium and motivating
- should not feel manipulative
- should help advanced users see a horizon beyond daily missions

==================================================
F. UI / UX MVP
==================================================

Build only minimal UI needed.

Possible surfaces:
- mastery indicators on skill screen
- prestige readiness / prestige marker on character summary
- arc stage display
- cycle/season progress card
- next major unlock card on home/profile

Requirements:
- dark premium
- command-center prestige
- serious, aspirational, endgame-worthy
- no childish fantasy UI
- no clutter

- keep schema changes additive and minimal
