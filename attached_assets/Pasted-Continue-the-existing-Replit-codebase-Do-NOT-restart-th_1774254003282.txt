1B. Wall Surfaces

BACK WALL:
This is where wall items live:

Zones: [Wall Art] [Window/Mirror] [Shelf] [Clock]
Items render directly on wall surface
Wall color/texture changes per room theme


LEFT WALL (angled, partial):
Creates depth illusion
Zones: [Side shelf] or decorative only
Width: 15% of canvas
Rendered at perspective angle

RIGHT WALL (angled, partial):
Same as left, mirror
Width: 15% of canvas
Zones: [Lighting fixture] or decorative

FLOOR:
Where all standing items live:
Desk, Plants, Character, Coffee machine, Audio
Floor material changes per room theme

CEILING (optional, top edge):
2% of canvas height
Subtle ceiling edge
Can show lighting effects from it

1C. Zone Reorganization by Surface

WALL ZONES (items on wall):
- wall_art — paintings, prints
- window — window/mirror
- shelf — floating shelf with books
- clock — wall clock

FLOOR ZONES (items on floor):
- desk — main work desk
- plants — floor plants
- audio — speaker system
- coffee — espresso machine

DESK SURFACE ZONES (items on desk):
- monitor — screen setup
- desk_lamp — task lighting
- desk_decor — small items

CEILING/OVERHEAD:
- overhead_light — pendant/chandelier

This creates a REAL room hierarchy.

PART 6 — LIGHTING SYSTEM
==================================================

Each room has dynamic lighting that changes
based on equipped lighting items:

DEFAULT LIGHTING:
Ambient: subtle room-specific glow
No strong light sources

LED AMBIENT (equipped):
Top ceiling edge: 2px LED strip SVG
Color: soft purple/blue by default
Casts downward glow gradient on back wall

ARC FLOOR LAMP (equipped):
Warm cone of light on floor
Radial gradient from lamp position
rgba(255,220,100,0.12) glow
Affects nearby area only

PENDANT LIGHT (new item for exec/penthouse):
Light source from ceiling center
Creates warm top-down illumination
rgba(255,200,80,0.08) full room tint

Lighting system implementation:

Each lighting item exports a LightEffect component
RoomCanvas accepts activeLight prop
LightEffect renders as absolute overlay
on room canvas with appropriate blend mode
Use: opacity blend (not multiply - too dark)

PART 10 — BACKEND UPDATES
==================================================

New API endpoints:

GET /api/room/environments
Returns all environments with user's unlock status

POST /api/room/purchase-environment
Body: { environmentId }
Deducts coins, unlocks environment
Returns: updated wallet, unlocked environments list

POST /api/room/set-environment  
Body: { environmentId }
Sets active environment (must own)

GET /api/room/state
Updated to include:
  activeEnvironment: environmentId
  placedItems with surface info: 
    { slot, surface, itemId }

New schema:
room_environments table:
  - id, slug, name, requiredLevel, price
  - isDefault (starter = true)
  
user_room_environments:
  - userId, environmentId, purchasedAt

Update room_state:
  - Add: activeEnvironmentId

Seed 6 environments on startup.

TECHNICAL REQUIREMENTS
==================================================

Room Canvas rendering:
  - Use React Native Views for surfaces (walls/floor)
  - SVG items positioned absolutely within surfaces
  - Perspective effect via:
    transform: perspective(800px) rotateX for floor
    OR use calculated positions to simulate perspective

Performance:
  - Canvas: useMemo for static backgrounds
  - Items: React.memo per item
  - Animations: Reanimated 2 only
  - No re-render when shop panel scrolls

Environment SVG previews:
  - Each environment has a small preview SVG
  - Used in selection screen cards
  - Pre-rendered, no state dependency
