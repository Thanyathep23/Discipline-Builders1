Goal:
Build a complete premium Wardrobe system accessible from the 
Game Mode section on the Rewards screen. This replaces the 
current placeholder "try again" state.

The Wardrobe must feel like a luxury fashion boutique and 
watch collection combined — designed specifically for 
entrepreneurs who appreciate premium lifestyle brands.

LEGAL REQUIREMENT:
Do NOT use real brand names, logos, or trademarks such as 
Rolex, AP, Patek Philippe, Richard Mille, Gucci, Louis Vuitton, 
or any other trademarked brand. Use premium inspired naming 
instead. See the catalog defined below.

==================================================
SYSTEM ARCHITECTURE OVERVIEW
==================================================

The Wardrobe system has 4 sections:

1. WATCHES — luxury timepiece collection
2. CLOTHING — premium apparel collection  
3. ACCESSORIES — lifestyle accessories
4. EQUIPPED — currently active items on character

Access: From Rewards screen → Game Mode → Wardrobe button
Unlock gate: coin balance + character level requirement
Effect: equipped items visually affect the Character Status Hub

==================================================
PART 1 — WARDROBE SCREEN STRUCTURE
==================================================

1A. Entry Point

When user taps "Wardrobe" from Game Mode section:
- navigate to Wardrobe screen (not a modal)
- full screen with back navigation
- tab bar at top: Watches | Clothing | Accessories | Equipped

1B. Screen Header

- Title: "WARDROBE"
- Subtitle: current equipped count e.g. "3 items equipped"
- Coin balance chip (top right, same as current design)
- Character mini-preview showing current outfit if practical

1C. Section Layout

Each tab shows a 2-column grid of item cards.

Each item card shows:
- item visual (watch face / garment illustration)
- item name
- brand series name
- rarity chip: Common / Refined / Prestige / Elite / Legendary
- price in coins OR "Level X required"
- state chip: Locked / Available / Owned / Equipped

1D. Filter Bar (below tabs)

- All / Owned / Available / Locked
- Sort: Price / Level / Rarity

PART 5 — ITEM DETAIL BOTTOM SHEET
==================================================

When user taps any item card, open a bottom sheet with:

5A. Content
- large item visual (watch face / clothing render)
- item name + series
- rarity chip
- description text
- color/variant selector (swatches)
- stats:
  - Level required
  - Coin cost
  - Character effect description
  - Rarity

5B. Action states

If LOCKED (level too low):
- Show: "Reach Level X to unlock"
- Show XP progress toward that level
- Button: disabled "Locked" state

If AVAILABLE (level met, can afford):
- Show coin cost
- Button: "Purchase — X Coins" (primary)

If AVAILABLE but cannot afford:
- Show coin shortfall: "Need X more coins"
- Button: "Not Enough Coins" (disabled)
- Secondary: "Earn More Coins →" (links to Missions)

If OWNED but not equipped:
- Button: "Equip" (primary)

If OWNED and EQUIPPED:
- Button: "Unequip" (secondary)
- Show "Currently Equipped" chip

5C. Character preview
- show a small character preview wearing this item if practical
- or show item on mannequin/stand silhouette

==================================================

7B. Clothing connection
- equipped outfit changes the character body layer
- outfit tier (Common → Legendary) upgrades visible character class
- must be reflected on the Character Status Hub screen

7C. Status tier influence
- equipped items above Prestige rarity add small passive
  boost to Finance/Lifestyle dimension
- display small "equipped boost" indicator on Character Status Hub

7D. Sync rule
- character visual must update when wardrobe changes
- change should be immediate on equip/unequip
- persist server-side per user

8C. Accessory visuals

==================================================
PART 9 — ECONOMY AND UNLOCK RULES
==================================================

9A. Purchase rules (server-side enforced)
- user must meet level requirement
- user must have sufficient coin balance
- on purchase:
  - deduct coins from wallet atomically
  - create reward_transaction record (type: purchase)
  - add item to user inventory
  - return updated wallet balance and owned item state

9B. Equip rules (server-side enforced)
- user must own item before equipping
- only one item per slot at a time
- equipping does not cost additional coins
- unequipping is always free

9C. Default items
- Starter White Shirt: auto-owned by all users at signup
- Dark Denim Premium: auto-owned by all users at signup
- Starter Timepiece: auto-owned at signup

9D. Anti-abuse
- prevent duplicate purchase of same item
- prevent equipping non-owned items
- prevent client-side coin balance manipulation
- all state changes audit-logged
