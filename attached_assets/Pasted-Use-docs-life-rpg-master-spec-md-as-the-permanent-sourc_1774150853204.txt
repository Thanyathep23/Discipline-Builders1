==================================================
A. CORE PRODUCT FANTASY FOR THIS PHASE
==================================================

This phase must make the user feel:

“My character is changing because my real life is changing.”

The user should be able to see:
- stronger posture
- better composure
- higher-status presentation
- more refined lifestyle cues
- more elite/prestige cues over time

The visual evolution should feel:
- premium
- motivating

- game-like but not childish

Important:
This phase is NOT about letting the user manually customize everything.
This phase is about rendering real progression into visible character change.

==================================================
B. CHARACTER EVOLUTION STATE ENGINE
==================================================

Build a data-driven character evolution state engine.

Purpose:
Convert real player progression into a structured visual state.

The engine should consume trusted internal state such as:
- Fitness status / relevant skill
- Discipline status / relevant skill
- Finance / Lifestyle status
- Prestige / mastery / title markers where relevant
- overall status tier/class
- possibly current arc if useful
- possibly selected starter/default appearance config if needed

The engine should output a compact visual state model, for example:
- bodyToneState
- postureState
- outfitTierState
- grooming/refinementState
- accessoryPlaceholderState
- prestigeAccentState
- facialConfidenceState
- lifestyleSignalState
- environmentHarmonyState (if lightly connected to room preview)

Requirements:
- keep the visual state model deterministic and explainable
- keep it recomputable from trusted progression data
- do not make it depend on arbitrary client-side overrides
- keep it small and maintainable

==================================================
C. LAYERED CHARACTER RENDERER
==================================================

Build a layered renderer instead of a combinatorial asset explosion.

Purpose:
Make character evolution technically realistic to implement and scale.

Use a composable approach such as:
- layered SVG
- layered React/React Native visual components
- deterministic image/layer composition
- token-driven rendered figure system

Support at minimum these conceptual layers:
1. Base body/character
2. Body tone / health / energy layer
3. Posture / confidence layer
4. Outfit quality / refinement layer
5. Lifestyle signal layer
6. Prestige accent layer
7. Background or subtle environment sync layer if useful

Requirements:
- each layer should be able to evolve independently
- the renderer should not require separate art for every full combination
- keep performance reasonable for mobile
- keep the style aligned with the current premium pseudo-3D character direction
- keep the figure full-body and readable on mobile
