==================================================
A. MONETIZATION PRINCIPLES
==================================================

This product must monetize in a way that preserves trust, fairness, and long-term retention.

Core rules:
- do NOT sell fake progress
- do NOT sell proof approval
- do NOT sell reward multipliers that break the economy
- do NOT sell direct competitive unfairness
- do NOT make the game feel manipulative
- preserve the seriousness and premium feel of the product

Allowed premium value areas:
- cosmetic identity upgrades
- prestige presentation upgrades
- exclusive themes / room environments
- premium challenge packs / curated content packs
- convenience features
- deeper personalization
- advanced insights
- premium export/share presentation
- premium integrations if appropriate
- premium community/circle capacity or advanced controls if fair

==================================================
B. PREMIUM MEMBERSHIP / SUBSCRIPTION LITE
==================================================

Add a lightweight premium membership layer.

Purpose:
Give users a coherent premium tier with meaningful benefits.

Possible premium benefits:
- premium room/themes
- premium prestige/profile cosmetics
- premium share card styles
- premium challenge packs / content packs
- enhanced integrations / exports
- deeper analytics/insight summaries if already easy to connect
- expanded circle/community tools if fair
- extra customization, not raw power

Requirements:
- premium should feel aspirational and valuable
- premium should not grant unfair gameplay advantage
- premium entitlements must be enforceable server-side
- premium state should be inspectable
- premium UI should be elegant, not spammy

Support at least:
- free tier
- premium tier
- entitlement checks for premium-only surfaces/content

==================================================
C. PREMIUM COSMETIC / PRESTIGE OFFERS
==================================================

Add premium offers that are status/value-driven, not power-driven.

Examples:
- premium room themes
- prestige display accents
- premium title frames / profile cosmetics
- premium command center variants
- elite showcase styles
- premium milestone display surfaces

Requirements:
- cosmetics should integrate with the existing marketplace/world/profile systems
- ownership and premium access must be server-side enforced
- some items may be premium-exclusive
- some items may be purchasable with earned currency after premium unlock
- keep the system elegant and premium

==================================================
D. CONTENT / CHALLENGE PACK MONETIZATION LITE
==================================================

Add a monetization layer for curated content packs.

Examples:
- Focus Mastery Pack
- Trading Review Pro Pack
- Recovery Rebuild Pack
- Prestige Sprint Pack
- Deep Work Elite Pack

Requirements:
- premium content packs should feel like high-quality guided experiences
- packs can unlock curated challenges, themes, cosmetic rewards, or advanced mission sets
- packs should use the existing mission/pack/event architecture where possible
- pack ownership must be tracked and enforceable
- do not create bloated DLC infrastructure

Rules:
- premium content should add depth and curation, not raw unfair advantage
- free users should still have a strong core experience
- premium packs should feel like “better crafted paths,” not “buy power”

==================================================
E. CONVENIENCE / VALUE OFFERS (NON-PAY-TO-WIN)
==================================================

Add only tasteful, fair convenience/value offers.

Possible examples:
- extra save/loadout slots for room/profile/showcase
- advanced export formats
- premium planning templates
- premium calendar/integration limits
- premium archive/history depth
- premium admin-like self-inspection summaries for personal use
- premium circle customization or extra private slots if fair

Requirements:
- convenience offers must not destroy fairness
- core gameplay loop remains intact for free users
- offers should feel like enhancement, not coercion
- no hard paywalls around the product’s basic promise

==================================================
F. OFFER / ENTITLEMENT ARCHITECTURE
==================================================

Build a lightweight entitlement system.

Requirements:
- track premium state
- track owned premium items/packs
- enforce access server-side
- expose entitlement status to the UI safely
- support:
  - active subscription/premium membership
  - owned premium packs
  - premium-exclusive cosmetics/themes
- keep implementation minimal and extensible

Suggested concepts:
- products
- offers
- entitlements
- purchase records
- subscription status
- premium flags
- offer visibility rules

Do NOT build a giant billing abstraction unless needed.

==================================================
G. PAYWALL / UPGRADE SURFACES
==================================================

Add minimal, high-quality premium surfaces.

Possible surfaces:
- premium overview page
- upgrade modal/screen
- premium badges/markers in marketplace
- locked premium theme preview
- upgrade CTA in premium-only sections
- premium comparison card (Free vs Premium)

Requirements:
- dark premium
- aspirational, sharp, calm
- no spammy popups everywhere
- paywall should explain value clearly
- should reinforce why premium exists and what it unlocks
- should preserve trust and seriousness

==================================================
H. ADMIN / INTERNAL OFFER CONTROLS
==================================================

Add minimal internal controls for offers.

Possible controls:
- activate/deactivate offers
- mark products as premium-only
- price/config fields if appropriate
- inspect entitlement assignments
- inspect premium content pack usage
- limited-time premium offers if already easy to support

Requirements:
- admin-only
- lightweight
- use existing admin/tuning surfaces where possible
- keep offer operations auditable

- preserve auditability
