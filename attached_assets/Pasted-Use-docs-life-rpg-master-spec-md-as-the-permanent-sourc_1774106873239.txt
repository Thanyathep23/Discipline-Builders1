==================================================
A. SMART ONBOARDING GUIDANCE LAYER
==================================================

Upgrade onboarding from “profile setup” into “guided entry into the game.”

Purpose:
Help users understand:
- what this product is
- what they should do first
- what matters right now
- why the system is asking for certain actions
- how progress will change their character/life/game state

Build guidance for these key moments:
1. first app entry
2. first Quick Start completion
3. first mission creation
4. first AI mission appearance
5. first focus session
6. first proof submission
7. first reward/result
8. first store/world/status interaction
9. first circle/community/invite interaction if relevant

Requirements:
- keep onboarding progressive, not overwhelming
- do not dump all instructions at once
- make guidance contextual and stage-based
- preserve existing onboarding/profile data structures where possible
- keep copy clear, premium, and non-corporate

==================================================
B. CONTEXTUAL IN-APP GUIDANCE
==================================================

Add smart contextual guidance across the app.

Purpose:
Reduce friction exactly where the user needs help.

Examples of surfaces that should get contextual guidance:
- Home / Command Center
- Mission Board
- AI mission cards
- Focus session screen
- Proof submission screen
- Reward/result screen
- Character evolution/status screen
- Marketplace/storefront
- Inventory/equip/apply flow
- Room/world screen
- Circle/community or showcase surfaces if already present

Possible forms of guidance:
- concise tip blocks
- contextual helper cards
- “why this matters” microcopy
- next action CTA
- explanation of what changed after an action
- beginner hint vs advanced hint modes if appropriate

Requirements:
- guidance should appear when useful, not constantly
- do not create noisy tutorial spam
- users should be able to dismiss or move past guidance
- guidance should adapt to user state where practical

==================================================
C. NEXT-BEST-ACTION SYSTEM
==================================================

Add a simple but intelligent “what to do next” layer.

Purpose:
Prevent users from feeling lost in a system with many surfaces.

Build a lightweight recommendation system for next actions.

Examples:
- complete Quick Start
- accept your first AI mission
- finish your first focus session
- submit proof for your completed mission
- equip your new identity item
- view your status evolution
- start your first room/world upgrade
- continue Standard/Deep Profile
- complete your active chain step
- claim or review a reward/unlock
- return via a comeback mission after inactivity

Requirements:
- next-best-action should be based on current user state, not static rules only
- at minimum, use clear rule-based logic if that is the safest path
- next action should be shown in one or more high-value surfaces:
  - home screen
  - profile/status
  - post-reward/result
  - onboarding continuation areas
- recommendations should feel useful, not generic filler

==================================================
D. EMPTY-STATE / FIRST-TIME / STUCK-STATE UX
==================================================

Upgrade UX for situations where users do not naturally know what to do.

Handle states such as:
- no missions yet
- no AI missions accepted yet
- no proof submitted yet
- no purchases yet
- no displayed items yet
- no room/world customization yet
- no circles joined yet
- partially completed onboarding
- failed or abandoned focus attempt
- rejected proof
- reward disappointment
- skill not growing
- user inactive / comeback state

Requirements:
- empty states should teach, not just say “nothing here”
- stuck states should help recovery, not shame the user
- low-momentum states should encourage the next viable action
- use the existing design language and identity of the product

==================================================
E. LIGHTWEIGHT APP COACH SURFACES
==================================================

Add a minimal “app coach” layer.

Purpose:
Make the product feel intelligently supportive without becoming a giant chat-first AI system.

Possible coach surfaces:
- Today’s Guidance card
- Why this matters panel
- Next evolution hint
- “You are close to…” card
- Recovery suggestion card
- Suggested first step for new feature areas
- Smart reminder to finish an incomplete flow

Requirements:
- the coach should feel calm, sharp, and useful
- not like a nagging assistant
- not like a toy chatbot
- ideally reuse existing data:
  - arc
  - skills
  - current mission state
  - onboarding depth
  - world/status evolution
  - reward state
  - comeback state
- coach guidance should reinforce the product fantasy:
  “your real-life actions are shaping your character and world”

==================================================
F. GUIDANCE INTELLIGENCE / RULES
==================================================

Build the guidance layer using the lightest reliable intelligence model.

Preferred logic order:
1. deterministic rule-based guidance based on current user state
2. optional AI-generated micro-guidance only where low-risk and already practical
3. preserve graceful fallback if AI is unavailable

The guidance system should consider signals such as:
- onboarding completion stage
- whether the user has created a mission
- whether the user has finished a focus session
- whether a proof is pending
- whether a reward is unclaimed or unexplored
- skill weak zones
- active arc
- active chain
- inventory/world/status inactivity
- comeback/inactivity state

Keep this practical and explainable.

==================================================
G. PROGRESSIVE DISCLOSURE / COMPLEXITY CONTROL
==================================================

The app is now deep. Add UX structure so users are not overwhelmed.

Requirements:
- avoid showing every system at equal priority all the time
- create clearer “what matters now” hierarchy
- advanced systems should open when relevant, not all at once
- preserve product depth, but stage user exposure intelligently

Examples:
- basic user sees essential actions first
- deeper systems such as world, prestige, cycles, or advanced store/item logic can be introduced at the right time
- use subtle unlock/explanation patterns, not giant info dumps

==================================================
H. MINIMAL INTERNAL / ADMIN CONTROLS
==================================================

Add only minimal internal controls if strictly needed.

Possible examples:
- guidance flags/toggles
- onboarding step activation
- feature visibility state for new-user guidance
- coach card enable/disable states
- debug view of current next-best-action state

Requirements:
- admin-only
- lightweight
- use existing admin/tuning surfaces where possible
- do not build a large personalization console

==================================================
I. UI / UX DIRECTION
==================================================

Design direction:
- dark premium
- calm, intelligent, high-end guidance
- serious but approachable
- elegant “coach” feel, not loud tutorial energy
- every hint should feel useful and deliberate

The experience should feel like:
- a premium command center that understands where the user is
- a system that quietly helps the user move forward
- an app that becomes easier as it becomes deeper

Do NOT make the UI:
- noisy
- childish
- popup-heavy
- over-explanatory
- generic SaaS onboarding

==================================================
J. BACKEND / SECURITY RULES
==================================================

- any guidance state that depends on user progression must be derived server-side or from trusted internal state
- preserve JSON-only APIs
- preserve ownership checks
- do not trust client-provided “completion” state blindly
- keep schema changes additive and minimal
- preserve existing mission/proof/reward/economy/world/community/premium flows
- if guidance is cached/persisted, keep it recomputable and safe
