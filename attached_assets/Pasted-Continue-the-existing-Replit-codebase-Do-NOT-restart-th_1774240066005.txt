==================================================
SYSTEM A — FOCUS MODE APP BLOCKING
==================================================

Platform reality:
- iOS: cannot truly block other apps (Apple restriction)
- Android: can detect app switching via AppState
- Both: can track when user leaves the app during focus

Build what is actually possible:

A1. Focus Session Active State

When a focus session starts:
- set a global "focus_active" flag
- store: session_id, mission_title, end_time, strictness_mode
- make this accessible across the app

A2. App Leave Detection (both iOS + Android)

Use React Native AppState to detect when user 
leaves the app during an active focus session:

import { AppState } from 'react-native';

When AppState changes to 'background' during focus:
- record: left_at timestamp
- record: duration_away when they return
- if away > 2 minutes: log as "distraction event"
- if away > 5 minutes: log as "focus broken"

Store in focus_sessions:
- distraction_count (number of times left app)
- total_distraction_seconds
- focus_break_events: [{left_at, returned_at, duration}]

A3. Return Overlay (iOS + Android)

When user returns to app during active focus session:

Show a full-screen overlay with:
- Dark background matching app theme
- Current mission title: "You're working on: [mission]"
- Time remaining: "Focus ends in: 47:23"
- Distraction count: "You've left [X] times"
- Two buttons:
  - "Resume Focus" (primary) → dismiss overlay
  - "End Session Early" (secondary/destructive) 
    → confirm dialog → end session with penalty note

Overlay must:
- appear every time user returns from background
- not be dismissible by swiping
- show for minimum 3 seconds before Resume is tappable
- use design system tokens (dark premium style)

A4. Strictness Mode Integration

Strict mode:
- overlay shows for 5 seconds minimum
- after 3+ distractions: show warning
  "3 distractions recorded. This will affect your reward."
- after 5+ distractions: show stronger warning
  "Focus quality is low. Consider restarting this session."

Balanced mode:
- overlay shows for 2 seconds minimum
- warning after 5+ distractions only

A5. Distraction Impact on Reward

Pass distraction data to the judge/reward engine:

distraction_count affects reward_multiplier:
- 0 distractions: +0.1 bonus multiplier
- 1-2 distractions: no change
- 3-5 distractions: -0.15 multiplier
- 6+ distractions: -0.30 multiplier
- total_distraction_seconds > 50% of session: 
  flag as "low_confidence" session

A6. Focus Status Bar (while in focus)

When focus session is active and user is in the app:
Show a persistent thin banner at the top of every screen:

[🎯 Focusing: Trading Analysis — 47:23 remaining]

- height: 36px
- background: accent.primary at 90% opacity
- text: mission name + time remaining
- tap: navigates back to active focus screen
- must appear on Home, Missions, Rewards, Profile screens
- must NOT appear on the focus session screen itself

A7. What NOT to claim

Do NOT add code claiming to:
- block TikTok/Facebook/YouTube/Instagram directly
- modify iOS Screen Time
- kill other app processes

These are impossible without OS-level permissions
that app stores do not allow.

The honest version: 
"We track when you leave and hold you accountable."
This is actually more respectful and still effective.

B2. Smart Routing Logic

Before calling any AI, run rule-based pre-screening:

function preScreen(proof: ProofSubmission): PreScreenResult {
  
  // Rule 1: Empty submission
  if (!proof.summary && !proof.files?.length && !proof.linkUrl) {
    return { verdict: 'rejected', reason: 'no_proof', skipAI: true }
  }
  
  // Rule 2: Too short
  if (proof.summary && proof.summary.trim().length < 20) {
    return { verdict: 'rejected', reason: 'too_short', skipAI: true }
  }
  
  // Rule 3: Duplicate detection
  const textHash = hashText(proof.summary || '')
  const isDuplicate = await checkDuplicateHash(proof.userId, textHash)
  if (isDuplicate) {
    return { verdict: 'rejected', reason: 'duplicate', skipAI: true }
  }
  
  // Rule 4: Generic phrases detection
  const genericPhrases = [
    'i completed the task',
    'done',
    'finished',
    'i did it',
    'task complete',
    'completed'
  ]
  const normalized = proof.summary?.toLowerCase().trim()
  if (genericPhrases.includes(normalized)) {
    return { verdict: 'followup_required', reason: 'too_generic', skipAI: true }
  }
  
  return { skipAI: false }
}

If skipAI is true: return verdict immediately, save cost.
Estimated: 35-40% of submissions caught by rules alone.

B3. Provider Selection Logic

function selectProvider(proof: ProofSubmission, context: JudgeContext): string {
  
  const hasImage = proof.files?.some(f => 
    f.mimeType?.startsWith('image/'))
  
  // Has image: need vision capability
  if (hasImage) {
    if (AI_PROVIDERS.gemini_flash.available) return 'gemini_flash'
    if (AI_PROVIDERS.openai_mini.available) return 'openai_mini'
    return 'rules' // fallback: ignore image, judge text only
  }
  
  // Text only: use cheapest available
  if (AI_PROVIDERS.groq.available) return 'groq'
  if (AI_PROVIDERS.gemini_flash.available) return 'gemini_flash'
  if (AI_PROVIDERS.openai_mini.available) return 'openai_mini'
  
  // No API keys available
  return 'rules'
}
