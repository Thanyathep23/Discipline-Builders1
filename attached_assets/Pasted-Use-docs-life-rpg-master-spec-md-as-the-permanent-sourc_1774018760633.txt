==================================================
A. POSITIONING / LANDING SURFACES
==================================================

Create or improve lightweight growth-facing surfaces that explain the product clearly.

Primary product positioning:
This is not a generic productivity app.
This is a serious Life RPG with an AI Game Master that turns real-life progress into visible character growth.

Build:
- a lightweight landing/marketing page or equivalent entry surface
- a concise value proposition section
- a “how it works” section
- a “why it feels different” section
- a privacy/trust section
- a CTA path into signup / invite / beta access

Messaging priorities:
- real life -> game progression
- AI missions and fair proof-based rewards
- skill growth tied to actual behavior
- identity progression, not fake gamification
- private-first and non-creepy positioning

Requirements:
- copy should feel premium, sharp, and modern
- not overhyped or childish
- not vague corporate language
- should convert a cold user into understanding the product within seconds

Do NOT build a giant marketing site.
Keep it focused and high-conviction.

==================================================
B. INVITE / REFERRAL LOOP LITE
==================================================

Add a lightweight invite system.

Build:
- invite codes or referral codes
- invite link generation
- referral attribution on signup if feasible
- basic tracking of:
  - inviter
  - invitee
  - invite accepted / completed signup / activated
- lightweight reward or recognition for successful invites

Possible rewards:
- symbolic badge
- invite milestone title
- invite-only cosmetic/profile marker
- small controlled in-game bonus if safe and non-inflationary

Rules:
- do not create spam incentives
- do not create money-like referral abuse
- rewards should feel meaningful but controlled
- invite loop should remain private-first and tasteful

Requirements:
- preserve ownership/security
- invite attribution should be server-side
- avoid easy self-referral abuse if feasible
- keep implementation lightweight

==================================================
C. SHARE-TO-INVITE BRIDGE
==================================================

Connect existing shareable cards/surfaces to lightweight growth actions.

Examples:
- after exporting a share card, offer:
  - copy invite link
  - invite a friend
  - share “start your Life RPG” message
- profile showcase can expose a tasteful invite CTA
- milestone cards can include subtle invite framing

Requirements:
- do not turn every share into spam
- share moments should remain identity-first, invite-second
- invite CTA should feel optional and elegant
- preserve privacy and avoid exposing sensitive data

==================================================
D. ACQUISITION ATTRIBUTION LITE
==================================================

Add simple growth attribution so the team can see what brings users in.

Track lightweight fields/events such as:
- acquisition source
- invite code used
- share card source if relevant
- landing CTA used
- signup source
- first activation source

Requirements:
- keep it privacy-safe
- keep it lightweight
- tie into existing telemetry where possible
- do not build a giant attribution system

Suggested source categories:
- direct
- invite
- share_card
- landing_page
- comeback_reentry
- manual/admin/test
- unknown

==================================================

==================================================

Add lightweight internal visibility for growth funnels.

Minimum funnels:
1. Landing page -> signup
2. Signup -> Quick Start complete
3. Quick Start -> first mission
4. Invite received -> signup
5. Invite signup -> activated user
6. Share card viewed/exported -> invite click/signup if feasible

Requirements:
- use existing admin/tuning surfaces where possible
- do not build a huge BI dashboard
- keep it tactical and lightweight
- make it possible to tell whether invites and positioning are working
