==================================================
A. RECOMMENDATION ENGINE CORE
==================================================

Build a lightweight, explainable recommendation engine.

Purpose:
Generate useful recommendations across the product without becoming a giant opaque system.

The engine should support recommendation types such as:
- next action
- next mission
- next skill improvement step
- next character/status evolution step
- next world/room upgrade step
- next store/product recommendation
- comeback/recovery recommendation
- next showcase/share-worthy milestone
- next premium/value suggestion if appropriate

Requirements:
- recommendations should be deterministic and explainable first
- use rule-based scoring and prioritization as the default
- optional AI microcopy can exist, but the underlying recommendation selection must stay understandable
- recommendations must be based on real user state
- recommendations should feel specific, not generic filler
- recommendations should not overwhelm the user with too many options

Suggested high-level inputs:
- onboarding/profile completion state
- current arc and arc stage
- weak and strong skills
- active missions and chains
- recent mission/proof outcomes
- focus session success/failure patterns
- inventory ownership and equipped state
- room/world/theme state
- current status tier
- current marketplace affordability and aspirations
- comeback/inactivity state
- premium entitlement state if relevant
- current cycle/season/endgame readiness if already available

==================================================
B. NEXT BEST ACTION SYSTEM V2
==================================================

Upgrade the existing “what to do next” system into a stronger recommendation layer.

Purpose:
Make the app feel like it knows the single most useful next step.

Requirements:
- choose a primary recommendation and optionally 1-2 secondary ones
- recommendations should be prioritized by impact, readiness, and user context
- the primary next action should appear in the highest-value surfaces:
  - home / command center
  - character/status screen
  - post-reward/result screen
  - onboarding continuation points
  - comeback/recovery states
- recommendations should adapt to user maturity:
  - new users get activation-focused next actions
  - intermediate users get progression-focused next actions
  - advanced users get mastery/prestige/world/economy-focused next actions

Examples:
- finish your first proof submission
- complete your active chain step
- improve fitness to unlock the next visible character evolution
- equip your newly unlocked status item
- upgrade your room theme to match your current prestige state
- buy the next most relevant lifestyle item you can actually afford
- complete a comeback mission before returning to harder challenges
- continue Standard/Deep Profile if it is blocking smarter personalization

==================================================
C. SMART MISSION RECOMMENDATIONS
==================================================

Add smarter mission recommendations.

Purpose:
Ensure the mission system feels intentionally guided, not just populated.

Build recommendations for:
- best next mission
- best next AI mission to accept
- best recovery/comeback mission
- best challenge/stretch mission when the user is ready
- best chain step to prioritize
- best skill-building mission for a weak zone

Requirements:
- mission recommendations should use existing mission, chain, arc, skill, and adaptive challenge data
- mission recommendations should be realistic for the user’s current state
- do not recommend missions that are obviously too hard or too easy
- if there are multiple good options, rank them cleanly
- explain briefly why a mission is recommended

Examples of explanation:
- “Best for raising Discipline without overwhelming your current recovery arc.”
- “Best next step to unlock your next Fitness visual evolution.”
- “Recommended because it fits your current Focus Recovery chain and available time.”

==================================================
D. SMART MERCHANDISING / STORE RECOMMENDATIONS
==================================================

Add intelligent store/catalog recommendations.

Purpose:
Help users see what item would feel most relevant, aspirational, or useful next.

Recommend items based on:
- current status tier
- current arc
- weak zones
- aspirational lifestyle direction
- current room/world state
- currently equipped items
- owned vs missing collection pieces
- affordability
- premium status if relevant
- prestige/mastery readiness if relevant

Examples:
- a better watch or accessory when status/lifestyle presentation is underpowered
- a room/world item that matches the user’s new arc stage
- a prestige display item after mastery progress
- an affordable upgrade that visibly improves the current character/world presentation
- a missing piece in a coherent identity set

Requirements:
- recommendations should not feel like random upselling
- recommendations should be aspirational but grounded in current state
- avoid pushing items the user cannot use or cannot afford unless clearly marked aspirational
- do not turn the app into a hard-sell storefront
- recommendation copy should feel elegant and tasteful

==================================================
E. SMART STATUS / WORLD PROGRESSION RECOMMENDATIONS
==================================================

Add recommendations that connect the user’s current progression to visible changes.

Purpose:
Make users understand how their next real-world actions can change what they see.

Examples:
- “Improve Fitness to unlock your next body/posture state.”
- “Raise Discipline to move from Hustle to Rising.”
- “Equip a higher-tier accessory to complete your current lifestyle presentation.”
- “Complete two more verified Trading missions to evolve your specialist identity layer.”
- “Upgrade your command center theme to match your current prestige level.”

Requirements:
- recommendations must bridge data and fantasy
- they should feel emotionally motivating
- they should be grounded in real thresholds or near-threshold states
- they should help users feel that progress is visible and near

==================================================
F. RECOMMENDATION EXPLANATION LAYER
==================================================

Recommendations should not feel magical or random.

Add a lightweight explanation layer:
- why this is recommended now
- what it helps improve
- what visible or meaningful result it could unlock
- how hard/long it is likely to be

Requirements:
- explanation must stay concise
- explanation should improve trust
- do not expose giant debug dumps
- explanation should sound premium and intelligent, not robotic

==================================================
G. PRIORITIZATION / RANKING MODEL
==================================================

Build a simple scoring/ranking model for recommendation selection.

The model should consider dimensions such as:
- readiness
- expected impact
- current relevance
- psychological appropriateness
- effort fit
- current momentum state
- affordability if store-related
- unlock proximity if progression-related

The ranking model should:
- prefer actionable over vague
- prefer relevant over flashy
- prefer sustainable over overwhelming
- avoid recommending too many things at once
- be explainable and tunable

Keep the system rule-based and transparent.
Do not build a giant ML pipeline.
