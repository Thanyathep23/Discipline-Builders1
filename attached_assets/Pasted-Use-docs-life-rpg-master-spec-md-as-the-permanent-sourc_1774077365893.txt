PHASE 17 SCOPE ONLY

Build only:

- full housing/world-builder
- redesign of the whole product

==================================================
STEP 1 — AUDIT FIRST
==================================================

Before changing anything:
1. inspect the current inventory lite, titles, badges, symbolic assets, shop items, reward transactions, wallets, prestige/mastery systems, and share/profile surfaces

3. identify what is missing for a meaningful buy/sell/equip experience
4. implement only the minimum set of systems needed
5. avoid broad refactors

==================================================
A. MARKETPLACE / STOREFRONT STRUCTURE
==================================================

Build a real game-storefront layer on top of the current lite systems.

Purpose:
Give earned currency meaningful sinks and make progression feel tangible.

Requirements:
- marketplace page / storefront structure
- item browsing by category
- featured items / recommended items if feasible
- item detail view
- purchase confirmation
- owned vs not-owned state
- price visibility
- affordability state
- clean empty/error states

Suggested marketplace categories:
- Titles
- Badges / Trophies
- Command Center / Room upgrades
- Profile cosmetics
- Prestige markers
- Utility-style symbolic unlocks
- Limited-time event items if already easy to connect

Rules:
- keep the storefront premium and game-like
- do not turn it into a cluttered ecommerce app
- use current dark premium / command-center style

==================================================
B. ASSET OWNERSHIP MODEL IMPROVEMENTS
==================================================

Strengthen the asset model so owned items feel real and usable.

Requirements:
- clear ownership state
- item type classification
- item rarity classification if useful
- acquisition source (purchase, mission, chain, event, milestone, prestige)
- active/equipped state where relevant
- owned history or acquisition timestamp if useful

Support at least these asset behaviors:
- own
- equip / set active (for titles, cosmetics, selected markers)
- display in inventory/profile
- mark limited or exclusive items where relevant

Do not overbuild a giant item system.
Keep it tactical and extensible.

==================================================
C. BUY / SELL / EQUIP / ACTIVE-USE FLOWS LITE
==================================================

Add the core economy actions.

Requirements:
- buy item with in-game currency
- equip/set active for supported item types
- unequip/switch active item
- optional lightweight sell-back flow or recycle flow if appropriate and safe
- if sell-back exists, keep values conservative and non-exploitable
- preserve transaction logging server-side
- all currency changes remain server-side only

Rules:
- prefer system-owned economy, not player-to-player trading
- if a sell-back mechanic exists, prevent abuse and reward inflation
- not every item must be sellable
- titles/badges/prestige unlocks may be non-sellable
- decorative/profile items can be equip-only
- some assets can be purchase-only, some unlock-only, some both

==================================================
D. ECONOMY BALANCE / ANTI-ABUSE RULES LITE
==================================================

Add only the minimum economy safety rules needed.

Requirements:
- server-side validation for purchases and sell-backs
- prevent duplicate purchases where item should be unique
- prevent race-condition double spends
- prevent negative balances
- prevent exploit loops with sell-back values
- keep reward and economy logs auditable
- keep rules explainable and simple

If needed:
- improve transaction metadata
- improve item uniqueness rules
- improve wallet consistency safeguards

Do NOT redesign the whole economy in this phase.

==================================================
E. INVENTORY / ITEM DETAIL / PROFILE CONNECTION
==================================================

Make purchased/unlocked assets visible and usable.

Requirements:
- inventory should clearly show:
  - owned items
  - equipped/active items
  - item source if useful
  - rarity/category
- item detail should explain:
  - what the item is
  - how it is obtained
  - whether it can be equipped, sold, or is permanent
- profile/character summary should reflect active equipped items where relevant
- marketplace and inventory should feel connected, not separate silos

Add only minimal internal tools if needed.

Possible controls:

- adjust item prices
- inspect purchase/sell activity
- inspect economy sink/source balance for major item classes

Requirements:
- admin-only
- use existing admin/tuning surfaces where possible
- keep tools tactical, not a giant economy console
